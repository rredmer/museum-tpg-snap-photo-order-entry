# SNAP Photographic Order Entry System

Technical Products Group SNAP 2010 Photographic Order Entry System

## Overview

The SNAP 2010 Photographic Order Entry System is designed to streamline the process of managing photographic orders. This system includes various modules for handling order entry, serialization, and drive control, among other functionalities.

## Installation

To install and run the SNAP 2010 Photographic Order Entry System, follow these steps:

1. Clone the repository:
    ```sh
    git clone https://github.com/your-repo/museum-tpg-snap-photo-order-entry.git
    ```
2. Navigate to the project directory:
    ```sh
    cd museum-tpg-snap-photo-order-entry
    ```
3. Open the project in Visual Studio.

## Usage

To use the SNAP 2010 Photographic Order Entry System:

1. Open the project in Visual Studio.
2. Build the project by selecting `Build` > `Build Solution`.
3. Run the application by pressing `F5` or selecting `Debug` > `Start Debugging`.

## Project Structure

The project is organized into the following directories:

- `Docs/`: Contains documentation files.
- `Src/`: Contains the source code for the application.
  - `Serializer/`: Contains files related to the serialization of application data.
  - `Shared/`: Contains shared modules and controls used across the application.
  - `Snap10/`: Contains the main application files.

## Modules

### Serializer

The `Serializer` module handles the serialization and deserialization of application data. Key files include:

- `AppSerializer.exe`
- `AppSerializer.vbp`
- `frmSerializer.frm`

### Shared

The `Shared` module contains shared controls and utilities used across the application. Key files include:

- `DriveControl.ctl`
- `ExplorerControl.ctl`
- `frmLogFile.frm`
- `frmSysInfo.frm`
- `modGetLicense.bas`
- `WiaCalls.bas`

### Snap10

The `Snap10` module contains the main application files. Key files include:

- `frmMain.frm`
- `frmSetup.frm`
- `SummaryControl.ctl`
